# PROJECT RULES

## Fonte da verdade

A pasta `/specs` é a única fonte oficial de requisitos.

Nenhuma funcionalidade deve ser implementada sem consultar esta documentação.

---

## Objetivo

O objetivo do projeto é disponibilizar materiais de eventos de forma elegante, organizada e extremamente simples.

---

## Filosofia

Sempre priorizar:

- simplicidade
- velocidade
- legibilidade
- organização

Nunca adicionar funcionalidades desnecessárias.

---

## Interface

Toda interface deve seguir estas características:

- minimalista
- moderna
- premium
- muito espaço em branco
- poucos elementos
- excelente responsividade

Inspirada no portal de materiais do Leandro Ladeira.

---

## Código

Sempre utilizar:

- TypeScript

- Componentes reutilizáveis

- Tailwind

- shadcn/ui

- Hooks quando necessário

Nunca duplicar código.

---

## Componentes

Todo componente deve possuir apenas uma responsabilidade.

Evitar componentes grandes.

---

## Organização

Nunca misturar:

- interface

- regras de negócio

- acesso ao banco

Cada responsabilidade deve possuir seu próprio arquivo.

---

## Antes de escrever código

Sempre:

1. Ler a pasta `/specs`.

2. Explicar a implementação.

3. Implementar.

4. Revisar.

---

## Após finalizar

Sempre sugerir melhorias, mas nunca implementá-las sem aprovação.
